# 儿童阅读冒险设计 Skill

`design-child-reading-adventures` 会先核验图书和版本证据，再把可确认的内容转化为适龄、儿童主导、真正依赖阅读的活动点子、成人阅读方案或完整阅读冒险手册。

它特别处理六类常见问题：不编造原书细节；没有准确分页就不写页码；游戏必须使用刚读到的内容；开放任务要让儿童真正改变目标、规则或结果；长线手册不能把同一种游戏换标题、道具或成品后重复使用；完整交付时将家长方案与孩子绘本式手册分开。

当前版本：`v0.1.1`
开源协议：Apache-2.0

## 能做什么

- `ideas`：少量阅读活动点子。
- `plan`：成人使用的阅读路线、节奏、活动和调整方案。
- `full`：分开的成人方案与儿童绘本式冒险手册。
- 核验作品、译者、版本、节选和页码证据等级。
- 设计推理、搭建、角色扮演、声音、移动和无痕思考活动。
- 输出 DOCX/PDF 时要求逐页检查可读性与碰撞。

## 快速安装

把 `design-child-reading-adventures/` 复制到：

```text
~/.codex/skills/design-child-reading-adventures/
```

刷新 Codex 后使用：

```text
请使用 design-child-reading-adventures，为一本儿童读物设计两周阅读冒险。
孩子9岁，每次30分钟，喜欢推理和手工。请先确认版本证据；没有准确分页时不要写原书页码。我要分开的家长方案和孩子手册。
```

详细步骤见 [INSTALL.md](INSTALL.md)。

## 核心原则

1. 安全、隐私、合法访问和版权优先。
2. 不从模型记忆补书中细节。
3. 活动必须通过“移除图书测试”：换成无关故事后若仍基本一样，就不是合格的核心活动。
4. 开放活动至少让儿童控制三个有实质影响的维度，并允许改变或放弃目标、规则或结果。
5. 今天只读、暂停、换门、不留作品，都算完成。
6. 长线手册按孩子实际的决策循环安排玩法节奏；8 站以上至少覆盖六种互动语法，且相邻站不重复同一种循环。

## 验证状态

- 固定基准：5/5 任务、19/19 必需标准通过；0 自动失败，0 回归失败。
- 定向回归：家长引导、成人／儿童分离、页码证据门、阅读依赖玩法、真正开放玩法与十站玩法节奏均通过。
- 完成真实 DOCX/PDF 双文件生成和逐页渲染检查；真实儿童信息与产物未进入本仓库。
- 已知限制：当前验证主要是静态／自审回归，尚未完成隔离的新代理 forward test。

## 安全、版权与隐私

- 只有核验同一版本分页后才写原书页码。
- 不复制大段受版权保护文本，不使用未经授权的整本图书。
- 不要求公开分享儿童姓名、学校、位置、照片或声音。
- 本 Skill 不保证学习结果，也不替代发展、特殊教育或安全方面的专业意见。

## 反馈
